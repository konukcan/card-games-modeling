What are abstractions?
Abstractions are compositional patterns that appear in multiple rules. In DreamCoder, these would be "learned" by the library learning component and added to the primitive vocabulary. Rules sharing abstractions are predicted to show transfer learning.

📖 Glossary & Interpretation Guide

Key Terms

Primitive
A basic function that cannot be decomposed further (e.g., get_suit, first)
Combinator
A function that combines other functions (e.g., map, filter, all)
Composition
The tree structure showing how a rule is built from primitives
Abstraction
A reusable pattern discovered across multiple rules
Level
The compositional complexity (0=atomic, 4=meta-combinator)
Base Rate
The probability that a random hand satisfies the rule
Recognition Network
Neural network that predicts which primitives are needed for a task

How to Interpret Results

  1. Transfer Prediction: Rules sharing abstractions (see Section 4) should show transfer. For example, learning Halves_copy_suits should help with Halves_copy_colors because both use halves_equal.
  2. Difficulty Prediction: Higher-level rules with more primitives should be harder to learn. Check the performance table (Section 5) to see if this holds.
  3. Curriculum Design: Start with Level 0-1 rules, then progress to higher levels. Within a family, order by base rate (extreme base rates first).
